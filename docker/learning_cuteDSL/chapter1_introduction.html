What is CuTeDSL?

CuTeDSL (CUDA Tensor Expression Domain Specific Language) is a high-productivity programming model developed by NVIDIA for writing high-performance CUDA kernels. It is part of the CUTLASS library and provides:

  • Tensor expressions for describing nested loop structures
  • Layout abstractions for managing memory access patterns
  • Warp-level operations for efficient tensor core usage
  • Automatic code generation for different GPU architectures

Why Use CuTeDSL?

  • Built-in pipeline abstractions
    • Traditional CUDA CuTeDSL
    Manual thread indexing Abstract tensor operations
    Explicit memory coalescing Layout-based memory access
    Complex warp synchronization
    Architecture-specific code Architecture-aware abstractions

    Key Concepts Overview

    1. Tensor

    The fundamental data structure representing multi-dimensional arrays.

    import cutlass.cute as cute

# Create a tensor from a PyTorch tensor
tensor = cute.make_tensor(torch_tensor)

    2. Layout

    Describes how data is organized in memory (stride, shape, swizzling).

    # Create a simple row-major layout
layout = cute.make_layout((128, 64))  # Shape (M, N)

    3. TiledMma

    Tensor Core matrix multiplication operation.

    tiled_mma = cute.make_tiled_mma(
    cute.MmaAtom.F16F16F16_TN,  # Atom operation
    cute.LayoutType.RowMajor,    # A layout
    cute.LayoutType.ColMajor     # B layout
)

    4. CopyAtoms

    Memory copy operations with vectorization.

    copy_atom = cute.make_copy_atom(
    cute.nvgpu.CopyUniversalOp(),
    cutlass.Float16
)

    Hello World: Simple Element-wise Operation

    Here's a minimal example showing CuTeDSL structure:

    import cutlass.cute as cute

class SimpleKernel:
    def __init__(self):
        self.threads_per_cta = 128
    
    @cute.kernel
    def kernel(self, input_tensor, output_tensor):
        # Get thread index
        tid = cute.arch.thread_idx()
        
        # Get block index
        bid = cute.arch.block_idx()
        
        # Compute global index
        idx = bid * 128 + tid
        
        # Perform operation
        if idx < output_tensor.size:
            output_tensor[idx] = input_tensor[idx] * 2.0
    
    def __call__(self, input_tensor, output_tensor):
        grid = (input_tensor.numel() + 127) // 128
        self.kernel(
            grid=(grid, 1, 1),
            block=[128, 1, 1],
            args=(input_tensor, output_tensor)
        )

    Architecture Support

    CuTeDSL supports multiple NVIDIA GPU architectures:

    • SM80 (Ampere) - Basic tensor core operations
    • SM90 (Hopper) - TMA, async copy, clusters
    • SM100 (Blackwell) - Enhanced TMA, 2-CTA MMA, TMEM
    Note: The examples in this tutorial are primarily for Blackwell (SM100) architecture, which offers the most advanced features.
