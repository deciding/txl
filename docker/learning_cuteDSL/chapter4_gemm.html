Overview

GEMM (General Matrix Multiply) is the most fundamental operation in deep learning. The CuTeDSL examples include several GEMM variants:

  • dense_gemm.py - Basic dense GEMM
  • dense_gemm_persistent.py - Persistent kernel for large matrices
  • dense_blockscaled_gemm.py - Block-scaled quantization GEMM
  • grouped_gemm.py - Variable-sized matrix batches

Key Components

1. Kernel Configuration

class DenseGemmKernel:
    def __init__(
        self,
        acc_dtype,          # Accumulator type (Float32, Float16, Int32)
        use_2cta_instrs,   # Blackwell 2-CTA MMA instructions
        mma_tiler_mn,      # MMA tile shape (M, N)
        cluster_shape_mn,  # Cluster dimensions
        use_tma_store,    # Use TMA for output
    ):
        self.acc_dtype = acc_dtype
        self.use_2cta_instrs = use_2cta_instrs
        self.cluster_shape_mn = cluster_shape_mn
        self.mma_tiler_mn = mma_tiler_mn
        self.use_tma_store = use_tma_store
        self.threads_per_cta = 128

GEMM Kernel Structure

Main Loop Pattern

@cute.kernel
def kernel(self, tiled_mma, tma_atom_a, tma_tensor_a, ...):
    # 1. Get thread/warp coordinates
    warp_idx = cute.arch.warp_idx()
    bidx, bidy, bidz = cute.arch.block_idx()
    
    # 2. Initialize accumulator
    acc = tiled_mma.make_fragment_C(shape)
    
    # 3. Main loop: load, compute
    for k_tile_idx in range(k_tile_cnt):
        # TMA load to shared memory
        cute.copy(tma_atom_a, gA, sA)
        
        # MMA computation
        cute.gemm(tiled_mma, acc, fragA, fragB, acc)
    
    # 4. Store result
    cute.copy(tma_atom_c, acc, gC)

Running GEMM

# Create kernel
gemm = DenseGemmKernel(
    acc_dtype=cutlass.Float32,
    use_2cta_instrs=True,
    mma_tiler_mn=(128, 128),
    cluster_shape_mn=(2, 2),
    use_tma_store=True
)

# Execute
gemm(a_tensor, b_tensor, c_tensor, stream)
Example command:
python dense_gemm.py --ab_dtype Float16 --c_dtype Float16 --acc_dtype Float32 --mma_tiler_mn 256,128 --cluster_shape_mn 2,1 --mnkl 8192,8192,8192,1 --use_tma_store --use_2cta_instrs
