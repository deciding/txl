Development Workflow

Step 1: Define Problem and Requirements

  • What operation? (GEMM, Attention, Norm, etc.)
  • Input/output shapes and data types
  • Target architecture (SM80, SM90, SM100)
  • Performance targets

Step 2: Choose Kernel Type

# Based on problem size and architecture
- Tiling strategy: CTA tile, warp tile, thread tile
- Persistent vs non-persistent
- Cluster-based (for large problems)

Step 3: Design Memory Hierarchy

# GMEM -> SMEM -> RMEM -> TMEM (Hopper+)
# Decide:
- What stays in registers vs shared memory
- Number of pipeline stages
- TMA usage (SM90+)
- TMEM usage (SM100)

Step 4: Configure MMA and Copy Atoms

# MMA Atom Selection
tiled_mma = cute.make_tiled_mma(
    cute.MmaAtom.F16F16F16_TN,  # Based on dtype
    layout_a,
    layout_b
)

# Copy Atom Selection
copy_atom = cute.make_copy_atom(
    cute.nvgpu.CopyUniversalOp(),
    dtype
)

Step 5: Implement Kernel Structure

@cute.kernel
def kernel(self, ...):
    # 1. Get coordinates
    tid = cute.arch.thread_idx()
    bid = cute.arch.block_idx()
    
    # 2. Initialize
    accumulator = tiled_mma.make_fragment_C(shape)
    
    # 3. Main loop
    for k in range(k_tiles):
        # Load
        cute.copy(tma_atom, gmem, smem)
        # Compute
        cute.gemm(tiled_mma, acc, fragA, fragB, acc)
    
    # 4. Store
    cute.copy(tma_atom_c, acc, gmem)

Step 6: Add Pipeline and Synchronization

# For overlapping loads and compute
pipeline = pipeline.PipelineTmaUmma.create(...)
producer_handle = pipeline.producer_acquire()
consumer_handle = pipeline.consumer_wait()

Step 7: Tune Performance

  • Adjust tile sizes
  • Adjust cluster shape
  • Adjust pipeline stages
  • Profile with Nsight Compute
