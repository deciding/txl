1. Overview: TMA Prefetch

The prefetch version adds explicit TMA prefetch hints to hide memory latency better than the basic persistent version.

FeatureDescription
Initial PrefetchPrefetch K tiles before mainloop starts
Rolling PrefetchPrefetch next tile while current tile computes
prefetch_distControls prefetch distance (0 = disabled)

2. Constructor with Prefetch Parameter

def __init__(
    self,
    acc_dtype: Type[cutlass.Numeric],
    use_2cta_instrs: bool,
    mma_tiler_mn: Tuple[int, int],
    cluster_shape_mn: Tuple[int, int],
    use_tma_store: bool,
    prefetch_dist: Optional[int] = None,
):
    self.acc_dtype = acc_dtype
    self.use_2cta_instrs = use_2cta_instrs
    self.cluster_shape_mn = cluster_shape_mn
    self.mma_tiler_mn = mma_tiler_mn
    self.mma_tiler = (*mma_tiler_mn, 1)
    self.use_tma_store = use_tma_store

    self.cta_group = (
        tcgen05.CtaGroup.TWO if use_2cta_instrs else tcgen05.CtaGroup.ONE
    )

    self.occupancy = 1
    self.epilogue_warp_id = (0, 1, 2, 3)
    self.mma_warp_id = 4
    self.tma_warp_id = 5
    self.threads_per_cta = 32 * len(
        (self.mma_warp_id, self.tma_warp_id, *self.epilogue_warp_id)
    )

    self.prefetch_dist = prefetch_dist

Functionality

Initializes kernel with optional prefetch distance parameter.

Shape Annotations

  • prefetch_dist: Optional[int] - None=auto, 0=disabled, >0=explicit

Why Like This

  • None defaults to num_ab_stage for optimal pipeline
  • 0 explicitly disables prefetch for debugging
  • >0 allows manual tuning of prefetch distance

3. Setup Attributes with Prefetch

def _setup_attributes(self):
    tiled_mma = self._create_tiled_mma()

    mma_inst_shape_k = cute.size(tiled_mma.shape_mnk, mode=[2])
    mma_inst_tile_k = 4
    self.mma_tiler = (
        self.mma_tiler[0],
        self.mma_tiler[1],
        mma_inst_shape_k * mma_inst_tile_k,
    )
    self.cta_tile_shape_mnk = (
        self.mma_tiler[0] // cute.size(tiled_mma.thr_id.shape),
        self.mma_tiler[1],
        self.mma_tiler[2],
    )

    self.cluster_layout_vmnk = cute.tiled_divide(
        cute.make_layout((*self.cluster_shape_mn, 1)),
        (tiled_mma.thr_id.shape,),
    )

    self.num_mcast_ctas_a = cute.size(self.cluster_layout_vmnk.shape[2])
    self.num_mcast_ctas_b = cute.size(self.cluster_layout_vmnk.shape[1])
    self.is_a_mcast = self.num_mcast_ctas_a > 1
    self.is_b_mcast = self.num_mcast_ctas_b > 1

Functionality

Computes MMA tile shape, cluster layout same as persistent version.

Shape Annotations

  • num_mcast_ctas_a: Number of CTAs to multicast A to
  • num_mcast_ctas_b: Number of CTAs to multicast B to

Why Like This

  • Multicast reduces memory traffic in cluster
  • Only enabled when cluster has multiple CTAs

4. Initial Prefetch Loop

prefetch_k_tile_cnt = self.prefetch_dist
if prefetch_k_tile_cnt is None:
    prefetch_k_tile_cnt = self.num_ab_stage - 2

if warp_idx == self.tma_warp_id:
    for k_tile_idx in cutlass.range(
        0, prefetch_k_tile_cnt, 1, unroll=1
    ):
        producer_handle = ab_producer.acquire_and_advance()

        cute.copy(
            tma_atom_a,
            tAgA_slice[(None, k_tile_idx)],
            tAsA[(None, producer_handle.index)],
            tma_bar_ptr=producer_handle.barrier,
            mcast_mask=a_full_mcast_mask,
        )
        cute.copy(
            tma_atom_b,
            tBgB_slice[(None, k_tile_idx)],
            tBsB[(None, producer_handle.index)],
            tma_bar_ptr=producer_handle.barrier,
            mcast_mask=b_full_mcast_mask,
        )

Functionality

Initial prefetch loads K tiles before the main computation loop starts.

Shape Annotations

  • prefetch_k_tile_cnt: Number of tiles to prefetch
  • producer_handle: Barrier handle for TMA operation

Why Like This

  • Prefetch fills pipeline before compute starts
  • num_ab_stage - 2 leaves slots for compute
  • Can be disabled by setting prefetch_dist=0

5. Main Loop with Rolling Prefetch

for k_tile_idx in cutlass.range(k_tile_cnt):
    if k_tile_idx < k_tile_cnt - prefetch_k_tile_cnt:
        producer_handle = ab_producer.acquire_and_advance(peek_ab_empty_status)

        cute.copy(
            tma_atom_a,
            tAgA_slice[(None, k_tile_idx + prefetch_k_tile_cnt)],
            tAsA[(None, producer_handle.index)],
            tma_bar_ptr=producer_handle.barrier,
            mcast_mask=a_full_mcast_mask,
        )
        cute.copy(
            tma_atom_b,
            tBgB_slice[(None, k_tile_idx + prefetch_k_tile_cnt)],
            tBsB[(None, producer_handle.index)],
            tma_bar_ptr=producer_handle.barrier,
            mcast_mask=b_full_mcast_mask,
        )

    if is_leader_cta:
        consumer_handle = ab_consumer.wait_and_advance(peek_ab_full_status)

        num_kblks = cute.size(tCrA, mode=[2])
        for kblk_idx in cutlass.range(num_kblks, unroll_full=True):
            kblk_crd = (None, None, kblk_idx, consumer_handle.index)

            cute.gemm(tiled_mma, tCtAcc, tCrA[kblk_crd], tCrB[kblk_crd], tCtAcc)
            tiled_mma.set(tcgen05.Field.ACCUMULATE, True)

        consumer_handle.release()

Functionality

Main loop with rolling prefetch - prefetches next tile while computing current.

Shape Annotations

  • k_tile_idx + prefetch_k_tile_cnt: Index of tile to prefetch
  • consumer_handle: Barrier for data ready

Why Like This

  • Rolling prefetch keeps pipeline full
  • prefetch_k_tile_cnt ahead of compute
  • Overlaps memory latency with computation

6. Prefetch Configuration Summary

prefetch_distBehavior
None (default)Auto: num_ab_stage - 2 tiles
0Disabled - no prefetch
>0Explicit prefetch distance

Key Takeaways

  1. Initial prefetch: Loads tiles before compute begins
  2. Rolling prefetch: Loads ahead while computing
  3. Tuning: prefetch_dist controls distance between load and compute
  4. Trade-off: More prefetch = better hiding but more SMEM
